# License Path Decision Packet

## Parent

Source-Wire Memory Engine Baseline Audit PRD.

## What to build

Create the license path decision packet for using `Source-Wire-Memory-Engine` as a runtime baseline without blurring the Apache-2.0 Source-Wire package boundary.

This slice is HITL because the license path affects long-term public adoption, contribution posture, and future runtime implementation.

## Acceptance criteria

- [ ] The packet compares AGPLv3 reference runtime, Apache-2.0 clean rewrite, dual-license owned code, and separate runtime package options.
- [ ] The packet explains implications for public adopters.
- [ ] The packet explains implications for future managed-hosted or SaaS paths.
- [ ] The packet explains implications for contribution acceptance.
- [ ] The packet recommends a default path for the next implementation unit.
- [ ] The packet lists legal review questions and unresolved risks.
- [ ] No runtime code is copied or implemented.
